Глава 3 Знакомство с блочной моделью

	В internet explorer есть ошибка, из-за которой элементы main отображаются как строчные, а не как блочные.

	-----------
	Пример регулирования ширины контейнера

	width: calc(30% - 3em);
	-----------

	По умолчанию box-sizing: content-box;(Растет наружу).

	Лучше использовать border-box (растет внутрь).

	Еще лучше использовать снипперт для включения наследования свойств box-sizing:

	:root{
		box-sizing: border-box;
	}
	*,
	::before,
	::after{
		box-sizing: inherit;
	}
	-------------

	Сделать зазор между блоками двумя способами:
		
		width: 29%;
		margin-left: 1%;

	или

		width: calc(30% - 1.5em);
		margin-left: 1.5em; 
	-------------

	Переполнение можно контролировать с помощю overflow:

	- visible (по умочанию) отображается все даже если за краями экрана;
	- Hiden скрывается;
	- Scroll;
	- Auto полосы прокрутки после переполнения.

	Можно управлять и горизонтальным переполнением:
	overflow-x 
	overflow-y
	-------------

	Указание высоты в процентах плохая идея тк высота зависит от контента, а браужер не всегда может распознать определение и будет игнорировать его.
	Лучше использовать vh.
	------------ 

	Для создания колонок одинаковой высоты можно использовать Css таблицы вместо flex:

	display:table; display: table-cell;

	display:table по умолчанию не будет расширятся до 100% ширины поэтому нужно width:100%;

	Пространство между ячейками border-spacing: x у;
	но это воздействует и на внешние края элементов, поэтому нужен новый обертывающий div с отрицательными полями, чтобы растянуть внешние края на величину border-spacing.

  3.2.4. Центрирование контента по вертикали.

  vertical-align ратотает только со строчными (в одной строке) и табличными элементами (внутри ячейки).

  Варианты выравнивания по высоте:

  - Использовать одинаковые верхние и нижние паддинги если высота контейнера не важна;
  - Если ипользование паддингов не возможно, то использовать свойства display: table-cell и vertical-aligne: middle к контейнеру;
  - Использовать flexbox верстку;
  - Если только она строка, то установить высоту строки равную высоте контейнера (может потребоваться inline-block);
  - Если извесна высота контента и контейнера центрировать контент с помощю абсолютного позиционирования;
  - Если высота контента неизвестна использовать абсолютное позиционирование в сочетании с трансформацией.

  3.3 Отрицательные значения полей. стр. 104

  Отрицательные значения верхних или левых полей отодвигают элемент влево или вверх. Добавление отрицательных значений для правых и нижних полей приводит к смещению последующих элементов влево и вверх.


  3.4. Схлопывание полей. стр. 106 

  Размер схлопнувшегося поля равен наибольшему из объединенных полей.

  Способы предотвращения схлапывания полей:

  - Применение свойства overflow отличное от visible;
  - Добавление границ или отступов между двумя полями;
  - Поля не схапываются вне плавающего контейнера, строчного блока или имеющего абсолютное или фиксированное значение;
  - В случае Flex верстки поля не схлапываются между элементами Flex макета. Это относится и к Css сеткам;
  - Элементы со значением table-cell table-row не имеют полей и не будут схлопываться. Исключения - элементы table, table-inline, table-caption.


  3.5. Расстояние между элементами в контейнере. стр. 109

  .button-link + .button-link{
	margin-top: 1.5em;
  }// + Комбинатор смежных элементов. Добавляет отступы только между элементами .button-link

  *+*{} - Селектор лоботомированой совы выбирает все элементы страницы, которые не являются первым дочерним элементом их родителя.


  Итоги главы:

  - Всегда работайте с border-box;
  - Избегайте установки явной высоты элемента;
  - Применяйте современные методы компановки тк display-table либо flex для создания колонок равной высоты или вертикально центрированного контента;
  - Если поля ведут себя странно, выполните нужные действия, чтобы предотвратить их схлапывание.
  - Попробуйте использовать на своей странице *+*{}.
