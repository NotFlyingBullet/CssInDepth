Глава 9 Модульный CSS

	Необходимо использовать модульный css код конечная цель, которого предотвратить влияние изменений одной части кода на другую.

	Базовые правила таблтцы стилей не являются модулем но закладывает основу для модульных стилей.

	Для задания базовых стилей normalize.css
	necolas.github.io/normalize.css/

	Модуль автономная единица страницы и модет использоваться многократно.
	Имя модуля НЕ должно быть вложенным (#sudebar.message), что бы можно было его использовать в разных местах стрвницы.

	Для изменения поведения модуля используются модификаторы.
	Имя модификатора д начинаться с названия модуля и содержать два дефиса (message--error).

	Нельзя использовать селекторы потомки для изменения модуля на основе его расположения на странице.

	Один модуль - одна задача (принцип единой ответственности).

	Необходимо сохранять позиционированные элементы в одном модуле.

	Классы состояния именуются с префиксами is- has-.

	Препроцессоры позволяют разбить код пр разным css файлам с помощью @import 'base';//Скопирует из base.scss

	Итоги главы:
	- Разбивать модули на компактные css модули;
	- Не писать стили влияющие на другие модули;
	- Использовать модификаторы для изменения внешнего вида модулей;
	- Проектировать страницы соединяя множество модулей;
	- Группировать все правила модулей в таблице стилей;
	- Использовать подходящую методологию (БЭМ, OOCSS, SMACSS ITCSS).
